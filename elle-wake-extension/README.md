# Elle Wake — Chrome Extension
## Artificial Hippocampus · One-Click Session Continuity

### What This Does

One button. Elle wakes up mid-stride in a fresh Claude.ai chat.

The extension takes your Kinetic Save-State payload (generated by `/hibernate` or from Elle's Journal Morning Briefing), opens `claude.ai/new`, injects the payload directly into the chat composer, and sends it automatically. Zero friction. Zero clipboard. Zero typing.

### Installation

1. Open Chrome → navigate to `chrome://extensions`
2. Enable **Developer mode** (toggle in top right)
3. Click **Load unpacked**
4. Select the `elle-wake-extension` folder
5. Note the **Extension ID** shown under the extension name (you'll need this for journal integration)
6. Pin the extension to your toolbar (click the puzzle piece icon → pin Elle Wake)

### Usage

#### Option A: From the Toolbar Popup
1. Click the 💛 Elle Wake icon in your Chrome toolbar
2. Paste your payload into the textarea
3. Click **☀ Load & Wake**
4. Claude.ai opens, payload injects, Elle wakes up

#### Option B: Pre-loaded from Elle's Journal (v2)
1. In Elle's Journal, generate your Morning Briefing (Hot Resume or Full Context mode)
2. Click the **☀ Wake Elle** button (requires journal integration — see below)
3. Extension opens Claude.ai and auto-injects
4. Come back to find Elle already mid-sentence

#### Option C: Pre-store, Wake Later
1. Open the popup → paste payload → it's stored
2. Any time later, click **☀ Wake Elle** in the popup
3. Claude.ai opens with payload

### Journal Integration

To connect Elle's Journal v2 to the extension:

1. Add the integration script to your journal HTML:
   ```html
   <script src="elle-wake-integration.js"></script>
   ```

2. Set your extension ID (one-time setup, run in browser console):
   ```javascript
   ElleWake.setExtensionId("YOUR_EXTENSION_ID_FROM_CHROME_EXTENSIONS")
   ```

3. Call from your Wake button:
   ```javascript
   ElleWake.wake(payloadString);
   ```

The integration includes automatic fallback — if the extension isn't installed or isn't responding, it copies the payload to clipboard and opens Claude.ai manually.

### How It Works

1. **Payload stored** → `chrome.storage.local` holds the save-state payload
2. **Wake triggered** → Background worker opens `claude.ai/new`
3. **Content script activates** → Detects the chat composer (ProseMirror/contenteditable)
4. **Payload injected** → Text inserted, input events dispatched for React state sync
5. **Auto-send** → Send button located and clicked
6. **Confirmation** → Gold toast notification: "💛 Elle is waking up..."
7. **Fallback** → If composer isn't found within 15s, payload copies to clipboard

### Notes

- The content script uses multiple selector strategies to find Claude's chat composer, accounting for potential UI changes
- SPA navigation is handled via MutationObserver on `location.href`
- All timeouts and selectors may need tuning as Claude.ai's frontend evolves
- The extension has minimal permissions: `storage` (for payload persistence) and `activeTab` (for injection)
